The 7‑stage pipeline

Stage 1
Scout Scrapes trending developer tools, popular GitHub repositories, and Product Hunt launches. Builds a real‑time picture of what's getting attention in a given space.
Stage 2
Analyze Maps the scouted tools against an existing portfolio. Finds the gap: what's missing, what's underserved, what a new tool could do better or more narrowly.
Stage 3
Score Ranks the candidates by buildability, market size, and differentiation. Picks the best bet from the shortlist.
Stage 4
Spec Writes a full product requirements document: features, user flows, API design, data model. The spec drives the build stage without ambiguity.
Stage 5
Build Scaffolds and implements the tool from the spec. Not a skeleton. A working product. Uses pre‑built templates to minimize token usage and stay on scope.
Stage 6
Test Verifies the tool actually works. Runs the core flows, checks edge cases, confirms the output matches the spec. If it doesn't pass, it doesn't ship.
Stage 7
Ship Commits the code, creates a GitHub repository, pushes, and opens a PR. Everything is documented and reviewable from the first commit.

Under the hood

Crash‑resistant state Pipeline state is persisted to disk after every stage. If a run fails at stage 5, the next run picks up exactly where it left off. No re‑scouting, no re‑speccing.
Streaming progress UI The web UI streams stage output in real time via SSE. You can watch what's being built, catch issues early, and intervene before the full pipeline completes.
Pre‑built templates Common patterns like Flask apps, CLI tools, and API wrappers are templated. The build stage fills in the specific logic rather than generating boilerplate from scratch. Faster and more consistent.
CLI and web UI Run it headless from the terminal with python factory.py, or use the web interface at port 5025 for a live progress view. Both modes use the same underlying pipeline.
